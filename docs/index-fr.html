Features

All-in-One Script

No piecemeal installs, no manual configs. The script handles everything.

🌐

Multi-site WordPress

Create unlimited websites on a single VPS. Auto-installs WordPress, creates database, configures Nginx, SSL — just enter the domain name.

📋 What each option does

✅ Add new website — Enter domain name → auto-creates folder, database, WordPress install, Nginx config. Nothing else is touched.

⚠️ Remove website — Creates a full backup FIRST (files + database), then removes Nginx config. Backup saved at /backup/removed/. Your data is NOT lost.

✅ Add redirect — Redirects one domain to another (e.g. old-site.com → new-site.com). No data is modified.

✅ Fix permissions — Resets file ownership and chmod (755 folders, 644 files). Does NOT delete anything.

✅ Maintenance mode — Toggles a temporary maintenance page. One click ON, one click OFF.

✅ Plugin management — List, enable, or disable plugins by renaming folders. No plugins are deleted.

🗄️

Per-Table DB Backup for Multi-GB NEW

Dumps each MySQL table individually for heavy databases. SHA256 checksum, 3 retries on failure. Battle-tested: 119 tables, 44MB (583MB uncompressed) — dump 4s, restore 26s, 100% accurate.

📋 What each option does

✅ Automatic (cron) — Runs daily at 3AM. Dumps each table as a separate .sql.gz file. Generates SHA256 checksum for verification. Retries 3 times on failure. Old backups auto-cleaned after 7 days.

✅ Manual run — Same process, triggered from menu. You can watch progress in real-time.

✅ Safe design — Uses --single-transaction so the database stays online during backup. No downtime, no data loss, no locking.

🔄

WP Auto-Update + Rollback NEW

Full backup before updating. Updates core, plugins, themes for all sites. Verifies HTTP status post-update — auto-rollback if site errors. Results sent via Telegram.

📋 What each option does

Step 1: ✅ Creates full backup (files + database) BEFORE any update

Step 2: ✅ Updates WordPress core, all plugins, all themes for each site

Step 3: ✅ Checks HTTP status after update — is the site still responding 200?

Step 4: ⚠️ If site returns error → auto-rollback from the backup created in Step 1

Step 5: ✅ Sends results summary to Telegram (success/failure per site)

🛡️ Your site is NEVER left broken — the rollback is automatic.

📟

Resource Alert + Telegram NEW

Monitors RAM, Disk, CPU, Swap with custom thresholds. Telegram alerts automatically when limits exceeded, 30-minute cooldown to avoid spam. 1-click cron setup.

📋 What each option does

✅ View current status — Shows RAM %, Disk %, CPU load, Swap, top processes. Color-coded: 🟢 normal, 🟡 warning (>70%), 🔴 critical (>90%).

✅ Telegram alert (cron) — Runs every 5 min. Only sends alert when threshold exceeded. 30-min cooldown prevents spam. Just monitoring — changes NOTHING on the server.

🧹

Smart Disk Cleanup NEW

Dry-run first to preview what will be cleaned. Removes old logs, journal, package cache, WP revisions, transients, spam comments. Shows exact space reclaimed.

📋 What each option does — does NOT delete your website!

🛡️ This feature only cleans junk files. It does NOT delete your WordPress sites or posts.

✅ [1] Old logs — Deletes rotated log files (*.gz, *.1, *.2...) in /var/log. These are old copies, not active logs.

✅ [2] Package cache — Clears dnf/apt download cache. Like clearing browser cache — safe.

✅ [3] WP transients — Deletes temporary cache entries in WordPress database. WordPress recreates them automatically.

✅ [4] WP revisions — Deletes old draft versions of posts. Your PUBLISHED posts are untouched.

⚠️ [5] Old backups — Asks before deleting backups older than 30 days. You must confirm.

✅ [6] Temp files — Shows /tmp size. Does not auto-delete.

💡 Always runs dry-run first (preview) before asking to execute.

🔑

SSH Key Manager NEW

Add/remove SSH keys, validate format (RSA, Ed25519). Lockout prevention: won't disable password if no key exists. sshd_config backup + auto-rollback on validation error.

📋 What each option does

✅ Add SSH key — Paste your public key → validates format (RSA/Ed25519) → adds to authorized_keys.

✅ Remove SSH key — Select which key to remove from the list.

✅ Disable password login — Only works if at least one SSH key exists. Won't lock you out.

✅ Backup sshd_config — Always backs up before changes. Auto-rollback if SSH test fails.

🛡️ Lockout prevention: the script checks SSH connectivity before applying changes.

Domain Health Dashboard NEW

Single command displays a full dashboard: HTTP status, SSL expiry, TTFB, database size, disk usage — for all sites on the VPS. Color-coded for clarity.

📋 What each option does

✅ View only — This is a read-only dashboard. It checks and displays, but changes NOTHING.

Shows for each domain: HTTP status, SSL expiry date, TTFB speed, database size, disk usage.

Color-coded: 🟢 good | 🟡 expiring soon | 🔴 error or expired.

WordPress Staging NEW

Clone WordPress to staging.domain.com (files + DB). Auto search-replace domain in DB. Protected by .htpasswd. Safe staging deletion with regex validation.

📋 What each option does

✅ Create staging — Clones your site to staging.domain.com (separate copy of files + database). Your original site is untouched.

✅ Auto search-replace — Updates domain references in the staging DB so links work correctly.

✅ Password protected — Staging site is behind .htpasswd, not publicly accessible.

⚠️ Delete staging — Only deletes the staging copy. Original site is never affected.

Automated Backup

Daily database + config backups. Cross-VPS sync for redundancy. PHP file malware scanning. 7-day retention with auto-cleanup.

📋 What each option does

✅ Run full backup — Dumps all databases + copies config files to /backup/. Nothing is deleted or modified.

✅ List backups — Shows existing backup files with dates and sizes.

⚠️ Restore database — Imports a SQL file into a specific database. You choose which DB. Does NOT auto-overwrite.

✅ Sync to backup VPS — Copies backups to another VPS via rsync. Source backups remain intact.

Cluster VPS

Link multiple VPS servers into a cluster. Sync backups, configs, migrate sites between VPS nodes. Cluster-wide health checks from any node.

📋 What each option does

✅ Add/Remove VPS — Register or unregister a VPS node in the cluster. Requires SSH key setup.

✅ Sync configs — Copies Nginx configs and scripts to other nodes as backup reference. Does NOT auto-apply on remote.

✅ Sync backups — Copies backup files to all other nodes via rsync. Source remains intact.

✅ Migrate site — Copies (not moves) a site to another VPS: files + DB + Nginx config. Original site stays running.

✅ Health check — Shows RAM, Disk, Load for all nodes. Read-only.

12-Step Malware Scanner

Scans WPCode snippets, backdoors, hidden files, shell injection, uploads malware, wp-config tampering, cron injection, fake admin users — auto-removes threats.

📋 What each option does — does NOT delete WordPress!

🛡️ The scanner quarantines suspicious files — it does NOT delete WordPress core files.

✅ Scans for: backdoors, shell injections, hidden PHP files in uploads, wp-config tampering, fake admin users, suspicious cron jobs, WPCode snippet injections.

✅ Quarantine: Suspicious files are MOVED to /root/.quarantine/, not deleted. You can review and restore them.

✅ Report: Full scan report with findings. You decide what to do.

Firewall Hardening

Kernel tuning for SYN flood, UFW/firewalld auto-config, fail2ban with 5 jails, SSH hardening, Nginx DDoS protection, auto-update OS, change root password.

📋 What each option does

✅ Kernel tuning — Optimizes TCP/SYN flood protection parameters. Does not affect services.

✅ UFW/firewalld — Auto-configures firewall to allow SSH(22), HTTP(80), HTTPS(443). Blocks other ports.

✅ fail2ban — Sets up 5 jails (SSH, Nginx) to auto-ban brute-force IPs. Temporary bans only.

✅ SSH hardening — Disables root password login (only if SSH key exists), changes default settings.

✅ Nginx DDoS protection — Rate limiting for requests. Legitimate users unaffected.

🛡️ Backs up all configs before changes. SSH access is tested before finalizing.

Gestion Multi-IP

Multiple IPs on your VPS? Assign specific IPs to specific websites via nginx listen directive. View all IPs, bind/unbind, verify. Auto-backup + rollback.

📋 What each option does

✅ View all IPs — Lists all IP addresses on the VPS. Read-only.

✅ Bind IP to site — Assigns a specific IP to a website via Nginx listen directive. Auto-backup config before change.

✅ Unbind IP — Reverts to default (listen on all IPs). Auto-backup + rollback on error.

✅ Verify — Tests that the binding works correctly.

FAQ

Frequently Asked Questions

Is the script safe to use?

Yes. The entire source code is open, you can inspect it before running. The script doesn't send data externally, except for Telegram alerts (if you configure them).

Which operating systems are supported?

Rocky Linux 8/9 (recommended), AlmaLinux 8/9, Ubuntu 22.04, Ubuntu 24.04. VPS needs at least 2GB RAM and 10GB disk space.

How is this different from HocVPS Script?

VPS 1-Touch v2.1 is superior with 9 modules: per-table DB backup for multi-GB databases, WordPress auto-update with rollback, SSH key manager, domain health dashboard, staging site, analytics, resource alert Telegram. HocVPS Script doesn't have these features.

Do I need Linux expertise to use it?

No deep expertise needed. Just know how to SSH into a VPS. The admin menu is fully interactive with clear numbered options — just pick a number.

Can I migrate from HocVPS?

Yes. Just backup your data from the old VPS (database + wp-content), install the script on a new VPS, then import. The script has built-in SQL import in the Database menu.
